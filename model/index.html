Encapsulamento

Encapsulamento é um conceito em programação que se refere a esconder os detalhes internos de um objeto e expor apenas o que é necessário para o uso daquele objeto.
Em outras palavras, ele protege os dados de um objeto, permitindo o acesso e a modificação desses dados apenas por meio de métodos definidos. Isso ajuda a manter a integridade dos dados e a evitar alterações indevidas.

Controller

Um Controller (ou Controlador) é uma parte de um sistema que lida com a lógica de entrada do usuário e coordena como o sistema responde a essa entrada.
Ele atua como intermediário entre a interface do usuário (como uma página web) e a lógica de negócios ou banco de dados.
Em frameworks como o MVC (Model-View-Controller), o Controller decide o que fazer com os dados recebidos e qual resposta enviar de volta ao usuário.

Model-View-Controller(MVC)

O padrão MVC (Model-View-Controller) é uma arquitetura de software que separa a aplicação em três componentes principais, cada um com uma responsabilidade distinta. A relação entre essas classes é essencial para organizar o código de forma modular e manter a separação de responsabilidades.

Componentes do MVC:

  1. O que é os componentes do MVC:
    • Responsabilidade: Representa os dados da aplicação e a lógica de negócios. O Model é responsável por interagir com o banco de dados e manipular os dados conforme necessário.
    • Relação com as outras classes: O Model comunica-se diretamente com o Controller. Ele fornece os dados necessários ao Controller e atualiza os dados quando recebe instruções do Controller.
  2. View (Visão):
    • Responsabilidade: É a interface de usuário. A View é responsável por apresentar os dados ao usuário de maneira visual. Ela exibe as informações fornecidas pelo Model de forma que o usuário possa interagir com elas.
    • Relação com as outras classes: A View recebe dados do Controller para ser exibida ao usuário. Ela depende do Controller para receber as informações que serão apresentadas na interface.
  3. Controller (Controlador):
      4. Responsabilidade: Atua como um intermediário entre o Model e a View. O Controller recebe as entradas do usuário (como cliques ou comandos), processa essas entradas (interage com o Model se necessário) e determina qual View deve ser exibida.
      Relação com as outras classes: O Controller manipula o Model em resposta às ações do usuário e, em seguida, escolhe a View apropriada para apresentar os resultados ao usuário. Ele é o componente que coordena a comunicação entre o Model e a View.

Getters e Setters

Getters e Setters são métodos especiais que permitem acessar e modificar os atributos de uma classe, mantendo o princípio do encapsulamento. Eles são usados para proteger os dados internos de uma classe, controlando como esses dados são acessados e modificados.

  • Getters: São métodos que "pegam" ou retornam o valor de um atributo privado de uma classe. Exemplo: getNome() retorna o valor do atributo nome.
  • Setters: São métodos que "setam" ou atribuem um valor a um atributo privado de uma classe. Exemplo: setNome(String novoNome) altera o valor do atributo nome.

Relação com as Classes

Dentro de uma classe, os atributos são geralmente definidos como privados (ou seja, não podem ser acessados diretamente de fora da classe). Para acessar ou modificar esses atributos, utilizamos getters e setters.

  • Encapsulamento: Ao usar getters e setters, você protege os dados internos de uma classe, pois controla como os atributos podem ser acessados e alterados. Isso ajuda a manter a integridade dos dados, já que você pode adicionar validações dentro dos setters, por exemplo, para garantir que o valor atribuído seja válido.

CRUD

é um acrônimo para as operações básicas de Create (Criar), Read (Ler), Update (Atualizar) e Delete (Excluir) que são realizadas em um banco de dados ou em objetos persistentes.

  • Create (Criar): Corresponde a adicionar novos dados (ou instâncias) a uma base de dados ou coleção.
  • Read (Ler): Corresponde a ler ou recuperar dados existentes.
  • Update (Atualizar): Corresponde a modificar dados existentes.
  • Delete (Excluir): Corresponde a remover dados existentes.

Relação entre Getters/Setters e CRUD

  • Create: Ao criar um objeto de uma classe, geralmente usamos o construtor para inicializar os atributos. Em alguns casos, podemos usar setters para configurar os valores após a criação do objeto.
  • Read: Para ler os valores dos atributos de um objeto, utilizamos getters. Esses métodos nos permitem acessar os dados encapsulados na classe.
  • Update: Para atualizar ou modificar os valores dos atributos, utilizamos setters. Eles permitem alterar o estado de um objeto de forma controlada.
  • Delete: Embora não diretamente relacionado a getters e setters, o conceito de exclusão geralmente envolve a remoção de objetos ou a alteração de estados que podem ser controlados por setters.
